arch-chroot
arch-chroot /mnt/debian /bin/bash
Dirancang untuk Arch → Arch
Direkomendasikan
sudo chroot
sudo chroot /mnt/debian /bin/bash --login
Stabil untuk cross-distro
Aspek arch-chroot sudo chroot
Mekanisme internal Menggunakan unshare -m untuk membuat mount namespace terpisah, lalu bind mount otomatis, lalu masuk chroot Murni masuk ke root baru tanpa namespace tambahan. Bind mounts dikelola secara eksplisit oleh pengguna sebelum perintah dijalankan
Bind mounts Otomatis: /dev /dev/pts /proc /sys /run. Namun karena namespace, visibility mount tidak selalu transparan dari host Manual dan eksplisit. Kamu tahu persis apa yang di-mount, urutan pemasangan, dan dapat verifikasi kapanpun dengan mount | grep /mnt/debian
Perilaku saat exit Berbahaya
Script cleanup berjalan otomatis untuk unmount semua bind. Jika ada proses Debian yang masih aktif, unmount gagal dan meninggalkan ghost mount
 Aman & terkontrol
Exit shell tidak memicu unmount apapun. Kamu yang menentukan kapan dan urutan unmount secara manual
Risiko "target is busy" Tinggi
Muncul ketika proses Debian (D-Bus socket, background daemon, shell yang tertinggal) masih aktif saat arch-chroot mencoba cleanup. Hasilnya: umount: /mnt/debian/dev: target is busy
 Rendah
Jika ada proses busy, kamu bisa identifikasi dan tangani terlebih dahulu dengan fuser -km /mnt/debian sebelum memulai unmount
Risiko korupsi filesystem Sedang–tinggi
Lazy unmount yang gagal sebagian dapat meninggalkan filesystem Debian dalam kondisi tidak konsisten, terutama jika terjadi di tengah operasi apt atau dpkg
 Rendah
Karena unmount dilakukan manual setelah semua operasi selesai dan terverifikasi, risiko filesystem korup jauh lebih kecil
Konteks desain asli Dibuat untuk instalasi Arch ke Arch. Mengasumsikan environment target adalah Arch dan menggunakan konvensi path serta init system yang sama Perintah POSIX standar. Tidak ada asumsi tentang distro target. Bekerja identik untuk Arch → Debian, Ubuntu, Void, atau distro apapun
Debugging saat gagal Sulit. Ghost mount tersembunyi dalam namespace. Sering harus reboot host Arch untuk bersih-bersih, yang sendiri berisiko jika Debian sedang dalam state tidak konsisten Mudah. Semua mount terlihat jelas di mount | grep debian. Proses yang menggunakan mount teridentifikasi via fuser -mv /mnt/debian
Kesederhanaan perintah Satu perintah. Lebih cepat dan terasa praktis di permukaan, tetapi menyembunyikan kompleksitas yang bisa meledak saat exit Memerlukan 5–7 perintah mount sebelum masuk. Lebih panjang, tetapi setiap langkah transparan dan dapat diverifikasi
Rekomendasi untuk Arch → Debian Hindari
Tool ini bukan untuk skenario ini. Risikonya nyata dan konsekuensinya sulit dipulihkan tanpa reboot
 Gunakan ini
Kontrol penuh, prediktif, tidak ada kejutan saat exit. Standar yang tepat untuk lintas distro
Anatomi masalah "target is busy" pada arch-chroot
Ketika kamu mengetik exit dari dalam arch-chroot, script cleanup berjalan otomatis mencoba unmount semua bind mount. Jika environment Debian pernah menjalankan daemon (D-Bus, udev, systemd unit), atau masih ada file descriptor yang terbuka, proses unmount gagal dengan pesan target is busy. arch-chroot kemudian melakukan umount -l (lazy unmount) sebagai fallback — ini mendetach mount point dari filesystem hierarchy, tetapi mount tetap aktif untuk proses yang masih menggunakannya. Hasilnya adalah "ghost mount" yang tidak terlihat tapi masih ada: perintah umount /mnt/debian/dev berikutnya gagal, sesi baru arch-chroot menumpuk mount baru di atas yang lama, dan jika ini terjadi di tengah operasi apt install atau dpkg, database paket Debian bisa korup.
Template sudo chroot yang benar untuk instalasi Debian dari Arch
# --- masuk ---
mount --bind /dev      /mnt/debian/dev
mount --bind /dev/pts /mnt/debian/dev/pts
mount --bind /proc     /mnt/debian/proc
mount --bind /sys      /mnt/debian/sys
mount --bind /run      /mnt/debian/run
cp /etc/resolv.conf /mnt/debian/etc/resolv.conf
sudo chroot /mnt/debian /bin/bash --login
# --- selesai, keluar dari chroot ---
exit
# --- verifikasi tidak ada proses aktif ---
fuser -mv /mnt/debian 2>/dev/null
# --- unmount urutan terbalik ---
umount /mnt/debian/run
umount /mnt/debian/sys
umount /mnt/debian/proc
umount /mnt/debian/dev/pts
umount /mnt/debian/dev
umount /mnt/debian
